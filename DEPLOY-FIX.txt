Fix: "The system cannot access the remote repository" on Afrihost cPanel
=====================================================================================

METHOD: HTTPS + GitHub Personal Access Token (works on shared hosting)


STEP 1 — Create a GitHub Personal Access Token
------------------------------------------------
1. Go to: https://github.com/settings/tokens
2. Click "Generate new token" → "Generate new token (classic)"
3. Note: "Afrihost deploy"
4. Expiration: pick 90 days or "No expiration"
5. Scopes: check ONLY "repo" (full control of private repositories)
6. Click "Generate token"
7. COPY THE TOKEN NOW — you won't see it again!
   It looks like: ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx


STEP 2 — Update the remote URL in cPanel
------------------------------------------
1. Log in to cPanel on Afrihost
2. Go to "Git Version Control"
3. Click "Manage" next to your joblynk repository
4. Change the remote URL to (replace YOUR_TOKEN with the token from Step 1):

   https://rinae-del:YOUR_TOKEN@github.com/rinae-del/joblynk.git

   Example:
   https://rinae-del:ghp_abc123xyz456@github.com/rinae-del/joblynk.git

5. Click "Update"


STEP 3 — Pull / Deploy
------------------------
1. Still in Git Version Control, click "Pull or Deploy"
2. It should now pull successfully


TROUBLESHOOTING
----------------
- If "Update" button is greyed out, delete the repo in cPanel and re-add it
  using the token URL from Step 2 as the clone URL
- If your GitHub username has special characters, URL-encode them
- Token expired? Repeat Step 1 to generate a new one
